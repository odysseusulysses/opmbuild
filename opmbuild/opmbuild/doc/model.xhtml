Object Model

The object model follows the OPM specification in the most part, i.e. one class per significant concept. There are then a set of interfaces and abstract classes to refer to super-classes of objects.

Primary Classes

These are the classes corresponding directly to concepts in the OPM specification.

  • Account: An annotable set of graph elements (artifacts, processes, agents, annotations).
  • Agent: An OPM agent graph element.
  • Annotation: A key-value annotation to an entity which can be annotated.
  • Artifact: An OPM artifact graph element.
  • Graph: An OPM graph.
  • Process: An OPM process graph element.
  • Time: A timestamp, either bounded by earliest-latest instants or a single instant.
  • Used: A 'used' edge between a process and an artifact.
  • WasControlledBy: A 'was controlled by' edge between a process and an agent.
  • WasDerivedFrom: A 'was derived from' edge between two artifacts.
  • WasGeneratedBy: A 'was generated by' edge between an artifact and a process.
  • WasTriggeredBy: A 'was triggered by' edge between two processes.

Abstract Classes

These are the abstract super-classes of the above primary classes, which are useful to refer to in some cases.

  • Annotated: An entity which is associated with a set of annotations, i.e. can be annotated.
  • Edge: Any kind of edge.
  • GraphElement: An element of a graph, i.e. node, edge or annotation.
  • Node: Any kind of graph node, i.e. agent, artifact or process.
  • RoleAnnotatedEdge: An edge which can be labelled with a role.
  • TimeAnnotatedEdge: An edge which can be labelled with a timestamp.
